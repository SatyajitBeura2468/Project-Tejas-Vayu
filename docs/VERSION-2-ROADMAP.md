# Version 2 roadmap

Version 2 should begin only after the experimental platform is stronger. The goal is not to decorate Version 1 with a “live” badge; it is to earn more precise claims through better instrumentation and repeatability.

## Phase 1 — Measurement foundation

- Preserve time-stamped raw readings in a documented, exportable format.
- Add temperature and humidity measurement.
- Replace or supplement the MQ-135 with more selective, calibrated instruments appropriate to the selected pollutant.
- Document warm-up, calibration, baseline correction and sensor replacement procedures.
- Record exact UV spectral output, power and chamber irradiance.

**Gate:** repeated blank, dark-control and activated trials with traceable raw data.

## Phase 2 — Experimental control

- Improve equal dosing and mixing between chambers.
- Control or record illumination, airflow, temperature and humidity.
- Randomise chamber assignment and run independent replicates.
- Define uncertainty, exclusions and a pre-declared analysis method.
- Evaluate intermediates and by-products, not only sensor decay.

**Gate:** reviewed protocol and reproducible comparison with uncertainty reporting.

## Phase 3 — Material engineering

- Compare coating concentration, binder, substrate preparation and curing.
- Test adhesion, abrasion, washing, fouling, weathering and long-term activity.
- Compare coating application with bulk incorporation where appropriate.
- Measure maintenance, reactivation and end-of-life requirements.

**Gate:** a safe candidate surface system with measurable durability.

## Phase 4 — Pilot deployment

- Start with a supervised school or hospital demonstration wall.
- Progress to sheltered transport structures, road dividers or other approved public surfaces.
- Include untreated matched controls and independent validation.
- Publish energy, cost, maintenance and environmental monitoring results.

**Gate:** ethical, permitted and independently reviewed outdoor evidence.

## Phase 5 — Connected public interface

- Build a secure ingestion service with device identity and schema validation.
- Separate raw, calibrated, quality-flagged and public data layers.
- Show last-updated time, sensor health, calibration state, missing intervals and provenance.
- Support offline buffering and never replace missing values with invented continuity.
- Provide downloadable datasets and versioned methodology.

Only at this phase should the website describe a dashboard as “live.”
